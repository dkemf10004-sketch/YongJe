## 추천 로직 변경

### 선턴 archetype 관련
- `HIGHROLLER_TURN_ONE` 상수 추가
  - core
  - openers
  - follow
  - support
  - blockers
  - observedPairs
- `turnOneArchetypeBonus(...)` 추가
- `turnOneReplicabilityPenalty(...)` 추가
- 추천 breakdown 항목에 아래 값 추가
  - `선턴 archetype`
  - `재현성 보정`

### 추천 제외 규칙 보강
- 이미 픽한 영웅이 추천에 다시 뜨지 않도록 여러 단계에서 차단 시도
- 프리밴/내 픽/상대 픽/최종 밴 영웅을 추천 후보에서 제외하도록 보강
- `allSelected()` 기반 잠금 처리 추가
- `isLockedHero(...)`, `isLockedHeroName(...)` 보조 함수 추가
- 추천 렌더 직전 `visibleSingles`, `visiblePairs` 필터 추가

### 카운터/패키지 점수 수정
- `counterBonus(...)` 방향 오류 수정
  - 상대가 후보를 카운터하면 감점
  - 후보가 상대를 카운터하면 가점
- 추천 사유 문구 수정
  - `상대 리스크` -> `상대 카운터 가능`
  - `상대 대응 가능` -> `상대에게 취약`
- `packageBonus(...)` 내부 카운터 판정 방향 오류 수정

### 조합 반영 관련
- `completionPressureBonus(...)` 추가
  - 실제 시너지 링크/패키지 연결이 있을 때만 작동
- 한때 `draftPhaseWeight(...)`로 전체 증폭을 넣었으나,
  데이터 근거 없이 무조건 변동을 키우는 방향이어서 제거함
- 현재는 데이터 근거가 있을 때만 추천 순위가 바뀌는 방향 유지

### 단계 기준 추천 수정
- `nextMyPickStage()`를 `state.stageIndex` 의존에서 분리
- 실제 보드에서 아직 비어 있는 `내 픽 단계`를 직접 찾도록 수정
- 선택/삭제/상대 입력 후에도 보드 상태 기준으로 추천 단계를 다시 잡도록 변경

## 영웅 데이터 수정
- `세뉴아` -> `제뉴아`
- `어둠 방벽 샬롯` -> `어린 여왕 샬롯`
- `디자이너 디리벳` 항목 삭제
- `월광 아라민타` 항목 삭제
- `백은칼날의 아라민타`와 `월광 아라민타`를 동일 영웅으로 보고 데이터 병합
- 최종 표기는 `백은칼날의 아라민타`로 유지

## 현재 남아 있는 핵심 이슈
- 사용자가 보고한 기준으로, `내 다음 선택 추천`이 여전히 보드 상태를 충분히 반영하지 않는 사례가 남아 있음
- 특히 아래 현상이 완전히 해결되었다고 확정할 수 없음
  - 프리밴 완료 직후 추천이 기대대로 변하지 않음
  - 내 1픽/상대 1~2픽 반영 후 추천 수치 변화가 거의 없음
  - 내가 고른 영웅이 여전히 추천 목록에 남는 재현 사례가 있음
- 현재까지는 추천 후보 잠금, 카운터 방향, 단계 선정 로직을 수정했으나,
  실제 재현 케이스를 완전히 잡으려면 디버그 패널로 내부 상태를 노출하는 추가 작업이 필요함
