## Epic Seven Draft Workbench Instructions

이 프로젝트의 HTML 드래프트 워크벤치는 에픽세븐 실시간 드래프트 보조 도구다.
에이전트는 아래 규칙을 기준으로 드래프트 상태를 해석하고, 추천/설명/수정 작업을 수행해야 한다.

### 인코딩 작업 지침
- 이 프로젝트의 HTML, JS, MD 파일은 모두 `UTF-8` 기준으로 읽고 쓴다.
- PowerShell 기본 인코딩이나 시스템 코드페이지(`EUC-KR`/`CP949`/`MS949`)를 신뢰하지 않는다.
- 한글이 포함된 파일을 수정할 때는 `Get-Content -Raw -Encoding UTF8`로 읽고, `Set-Content -Encoding UTF8`로 저장한다.
- 콘솔 출력에 한글이 깨져 보여도 파일 자체 인코딩이 유지되는지 별도로 확인한다.
- 파일 전체를 문자열 치환으로 다시 쓰는 방식은 인코딩 손상 위험이 크므로, 가능하면 `apply_patch`를 우선 사용한다.
- `shell_command`로 파일을 쓸 때는 반드시 인코딩을 명시하고, 수정 후 UTF-8로 다시 읽어서 검증한다.
- HTML 상단의 `<meta charset="UTF-8">`와 실제 파일 저장 인코딩이 항상 일치해야 한다.
- 한글이 많은 파일은 가능하면 부분 수정만 수행하고, 전체 재저장은 피한다.
- 인코딩이 의심되면 수정 전에 원본 파일을 UTF-8로 읽어 한글이 정상인지 먼저 확인한다.
- 깨진 문자열이 보이면 추가 수정 전에 인코딩 문제부터 해결하고, 깨진 상태의 텍스트를 기준으로 덮어쓰지 않는다.

### PowerShell 수정 원칙
- 파일 읽기 예시: `Get-Content -Raw -Encoding UTF8 <path>`
- 파일 쓰기 예시: `Set-Content -Encoding UTF8 <path>`
- 대량 치환이나 전체 저장이 필요하면 먼저 백업 가능 여부를 확인하고 진행한다.
- 콘솔 표시 문자열과 실제 파일 바이트는 다를 수 있으므로, 저장 후 재읽기 검증을 필수로 한다.
### 목적
- 프리밴, 픽, 최종 밴의 단계별 진행을 관리한다.
- 현재 보드 상태를 기준으로 다음 픽 추천과 최종 밴 추천을 계산한다.
- 추천은 단순 메타 점수만이 아니라 시너지, 카운터, 패키지 조합, 밴 노출 리스크까지 함께 반영해야 한다.

### 드래프트 진행 규칙

#### 선픽일 때 단계 순서
1. 내 프리밴 2명
2. 상대 프리밴 2명
3. 내 1픽
4. 상대 1~2픽
5. 내 2~3픽
6. 상대 3~4픽
7. 내 4~5픽
8. 상대 5픽
9. 내 최종 밴
10. 상대 최종 밴

#### 후픽일 때 단계 순서
1. 내 프리밴 2명
2. 상대 프리밴 2명
3. 상대 1픽
4. 내 1~2픽
5. 상대 2~3픽
6. 내 3~4픽
7. 상대 4~5픽
8. 내 5픽
9. 내 최종 밴
10. 상대 최종 밴

### 프리밴 규칙
- 내 프리밴 2명과 상대 프리밴 2명을 각각 입력한다.
- 실제 프리밴 적용 대상은 양쪽 프리밴의 합집합이다.
- 중복 프리밴은 표시하되, 실제 제외 수는 중복 없이 계산한다.
- 프리밴 단계에서는 이미 어느 한쪽 프리밴에 들어간 영웅을 다시 선택할 수 없다.

### 픽 규칙
- 실제 프리밴된 영웅은 픽할 수 없다.
- 이미 내가 픽했거나 상대가 픽한 영웅은 다시 선택할 수 없다.
- 각 팀의 세 번째 픽은 `뱅가드`로 간주한다.
- 뱅가드는 최종 밴 대상에서 제외된다.

### 최종 밴 규칙
- 내 최종 밴 단계에서는 상대 5픽 중 뱅가드를 제외한 1명을 밴한다.
- 상대 최종 밴 단계에서는 내 5픽 중 뱅가드를 제외한 1명이 밴된다.
- 이미 최종 밴으로 지정된 대상은 다시 선택할 수 없다.

### 추천 로직 원칙
에이전트는 추천이나 설명을 만들 때 아래 요소를 함께 고려해야 한다.

1. 메타 가치
- 승률
- 픽률
- 밴률

2. 아군 시너지
- 현재 내 픽과 직접 연결되는 영웅 우대
- 상호 시너지 관계가 있으면 추가 가점

3. 상대 카운터 관계
- 후보가 상대 조합을 견제하면 가점
- 상대가 후보를 강하게 카운터하면 감점

4. 패키지 조합 보너스
- 사전에 정의된 패키지 조합 축이 연결되면 큰 가점
- 1명 연결보다 2명 이상 연결을 더 높게 평가
- 상대 조합이 해당 패키지를 압박하면 보너스를 감쇠

5. 초반 선점 가치
- 초반 라운드에서는 범용성이 높고 확장성이 좋은 영웅을 우선 평가한다.

6. 긴급도
- 밴률이 높거나 공통 프리밴군에 속하는 영웅은 선점 가치가 높다.

7. 최종 밴 노출 리스크
- 최종 밴으로 잘릴 가능성이 큰 고밴률 영웅은 감점 요소다.

8. 디코이 가치
- 고밴률 영웅은 상대 밴을 유도하는 미끼 역할을 일부 인정할 수 있다.

9. 뱅가드 보호 가치
- 내가 이미 2픽을 완료한 상황이라면, 다음 픽은 뱅가드가 되므로
  시너지 축이나 핵심 패키지를 보호할 수 있는 영웅을 높게 평가한다.

### 2인 페어 추천 원칙
- 다음 단계가 2픽 구간이면 단일 추천 외에 2인 조합 추천도 계산한다.
- 두 영웅 사이 직접 시너지가 있으면 가점한다.
- 같은 패키지 조합 소속이면 추가 보너스를 준다.
- 초반 2픽 구간에서는 뱅가드 보호 가치도 일부 반영한다.

### 최종 밴 추천 원칙
에이전트는 내 최종 밴 추천 시 아래 우선순위를 따라야 한다.
- 상대 영웅이 내 조합을 얼마나 강하게 카운터하는지
- 상대 조합의 핵심 시너지 축인지
- 상대 패키지 조합의 연결 고리인지
- 일반 메타 위험도(픽률, 밴률)가 높은지

즉, `상대 승리 플랜을 가장 크게 줄이는 대상`을 우선 밴 후보로 본다.

### 공통 운영 원칙
- 추천 점수는 참고값이며, 실제 설명에서는 현재 보드 기준 시너지와 카운터 관계를 우선 해석한다.
- 초반에는 단독 성능보다 범용 선픽과 조합 확장성을 우선한다.
- 중반에는 내 조합 완성과 상대 핵심 견제를 동시에 본다.
- 후반에는 최종 밴 이후에도 조합이 남는지 확인한다.
- 최종 밴 직전에는 상대의 실질적인 승리축 제거를 우선한다.

### 라운드 전환 규칙
- `다음 라운드`가 실행되면 상대 프리밴과 상대 픽은 초기화된다.
- 내 프리밴은 현재 선/후픽 상태 기준으로 저장 및 복원된다.
- 여러 판 반복 시 내 프리밴 성향을 유지하는 용도로 해석한다.

### 에이전트 작업 지침
- UI 수정 시 단계 순서, 뱅가드 규칙, 최종 밴 제외 규칙을 깨뜨리지 말 것.
- 추천 로직 수정 시 메타 점수만으로 단순화하지 말 것.
- 패키지 조합, 특수 카운터, 초반 선점 가치, 최종 밴 노출 리스크를 유지할 것.
- 설명 문구를 생성할 때는 왜 추천되는지를 시너지, 카운터, 조합 연결 기준으로 짧고 명확하게 제시할 것.
- 사용자가 전략 설명을 요청하면 현재 보드 기준으로 다음 픽, 2인 페어, 최종 밴 관점까지 분리해서 설명할 것.
